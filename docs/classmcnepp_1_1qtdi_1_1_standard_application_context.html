bool setAsGlobalInstance ()
 Sets this ApplicationContext as the global instance.
Derived classes should invoke this method as the last line of their constructor. This method will set this as the global instance.
Why can this not be done automatically in ApplicationContextImplBase's constructor?
Unfortunately, this would violate the C++ standard. ApplicationContextImplBase's constructor runs before the constructor of the derived class. If it did set the global instance to this, a reference to an incomplete object would be accessible by external code via QApplicationContext::instance()

Detailed Description

A ready-to use implementation of the QApplicationContext. Using this class is the "canonical way" of instantiating a QApplicationContext.
It should be used in all places where you rely on what the interface QApplicationContext offers, without the need to add more functionality.
Should you want to provide a customized version of QApplicationContext, or should you want to augment your class with additional functionaly, you may want to resort to the helper-class ApplicationContextImplBase.

◆ resolveConfigValue()

virtual QVariant mcnepp::qtdi::StandardApplicationContext::resolveConfigValue ( const QString & expression)
overridevirtual

Resolves an expression.
Thread-safety: This function may be safely called from any thread.

Parameters
expressionwill be parsed in order to determine the QConfigurationWatcher::currentValue().
In case the expression is a simple String, it will be returned as is.
The expression may contain one or more placeholders which will be resolved using the underlying configuration. A placeholder is enclosed in curly brackets with a preceding dollar-sign.
"${name}" will be resolved with the configuration-entry "name".
"${network/name}" will be resolved with the configuration-entry "name" from the section "network".
"${host}://${url}" will be resolved with the result of the concatenation of the configuration-entry "host", a colon and two slashes and the configuration-entry "url".
The special character-sequence asterisk-slash indicates that a value shall be resolved in a section and all its parent-sections:
"* /network/hosts/${host}" will be resolved with the configuration-entry "name" from the section "network/hosts", or its parent sections.
Returns
a valid QVariant, or an invalid QVariant if the expression could not be parsed.

virtual QConfigurationWatcher * mcnepp::qtdi::StandardApplicationContext::watchConfigValue ( const QString & expression)
overridevirtual

Obtains a QConfigurationWatcher for an expression.
If autoRefreshEnabled() and the expression can be successfully parsed, this function returns an instance of QConfigurationWatcher.
Using the Q_PROPERTY QConfigurationWatcher::currentValue(), you can then track the current configuration.
Thread-safety: This function may be safely called from any thread. The returned QConfigurationWatcher's will have a thread-affinity to the QApplicationContext's thread.

Parameters
expressionwill be parsed in order to determine the QConfigurationWatcher::currentValue().
The expression shall contain one or more placeholders which will be resolved using the underlying configuration. A placeholder is enclosed in curly brackets with a preceding dollar-sign.
"${name}" will be resolved with the configuration-entry "name".
"${network/name}" will be resolved with the configuration-entry "name" from the section "network".
"${host}://${url}" will be resolved with the result of the concatenation of the configuration-entry "host", a colon and two slashes and the configuration-entry "url".
The special character-sequence asterisk-slash indicates that a value shall be resolved in a section and all its parent-sections:
"* /network/hosts/${host}" will be resolved with the configuration-entry "name" from the section "network/hosts", or its parent sections.
Returns
QConfigurationWatcher that watches the expression, or nullptr if the expression could not be parsed, or if auto-refresh has not been enabled.

QApplicationContext * newDelegate ( const QLoggingCategory & loggingCategory,
QApplicationContext * delegatingContext )
friend

Creates an ApplicationContext as a delegate for another context.
With the help of this function you can implement the interface QApplicationContext yourself.
The delegatingContext will be used for various purposes:

  • It will become the QObject::parent() of the delegate.
  • It will be injected into init-methods of services that choose to provide such a method.
  • It will be attached to all ServiceRegistrations and can be obtained via ServiceRegistration::applicationContext().
    Parameters
    loggingCategorywill be used for logging.
    delegatingContextthe context that will delegate its call to the delegate.

    an ApplicationContext that serves as the delegate for the delegatingContext.
