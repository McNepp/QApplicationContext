virtual bool mcnepp::qtdi::QConfigurationResolver::autoRefreshEnabled ( ) const
pure virtual

Has auto-refresh been enabled?
If enabled, you may use watchConfigValue(const QString&) in order to watch configuration-values.
Also, it is possible to use mcnepp::qtdi::autoRefresh(const QString&) to force automatic updates of service-properties, whenever the corresponding configuration-values is modified.
When using a StandardApplicationContext, auto-refresh can be enabled by putting a configuration-entry into one of the QSettings-objects registered with the context: 

[qtdi]
enableAutoRefresh=true
; Optionally, specify the refresh-period:
autoRefreshMillis=2000
Returns
true if auto-refresh has been enabled.

Retrieves a value from the ApplicationContext's configuration.
This function will be used to resolve placeholders in Service-configurations. Whenever a placeholder shall be looked up, the ApplicationContext will search the following sources, until it can resolve the placeholder:

  1. The environment, for a variable corresponding to the placeholder.
  2. The instances of QSettings that have been registered in the ApplicationContext.
    See also
    mcnepp::qtdi::config()
    Parameters
    keythe key to look up. In analogy to QSettings, the key may contain forward slashes to denote keys within sub-sections.
    searchParentSectionsdetermines whether the key shall be searched recursively in the parent-sections.
    Returns
    the value, if it could be resolved. Otherwise, an invalid QVariant.

◆ watchConfigValue()

virtual QConfigurationWatcher * mcnepp::qtdi::QConfigurationResolver::watchConfigValue ( const QString &  expression)
pure virtual

Obtains a QConfigurationWatcher for an expression.
If autoRefreshEnabled() and the expression can be successfully parsed, this function returns an instance of QConfigurationWatcher.
Using the Q_PROPERTY QConfigurationWatcher::currentValue(), you can then track the current configuration.

Parameters
expressionwill be parsed in order to determine the QConfigurationWatcher::currentValue().
The expression shall contain one or more placeholders which will be resolved using the underlying configuration. A placeholder is enclosed in curly brackets with a preceding dollar-sign.
"${name}" will be resolved with the configuration-entry "name".
"${network/name}" will be resolved with the configuration-entry "name" from the section "network".
"${host}://${url}" will be resolved with the result of the concatenation of the configuration-entry "host", a colon and two slashes and the configuration-entry "url".
The special character-sequence asterisk-slash indicates that a value shall be resolved in a section and all its parent-sections:
"* /network/hosts/${host}" will be resolved with the configuration-entry "name" from the section "network/hosts", or its parent sections.
Returns
QConfigurationWatcher that watches the expression, or nullptr if the expression could not be parsed, or if auto-refresh has not been enabled.
