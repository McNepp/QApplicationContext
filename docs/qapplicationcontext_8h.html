enum class mcnepp::qtdi::Kind
strong

Specifies the kind of a service-dependency. Will be used as a non-type argument to Dependency, when registering a service. The following table sums up the characteristics of each type of dependency:

 Normal behaviourWhat if no dependency can be found?What if more than one dependency can be found?
MANDATORYInjects one dependency into the dependent service.If the dependency-type has an accessible default-constructor, this will be used to register and create an instance of that type.
If no default-constructor exists, publication of the ApplicationContext will fail. 		Publication will fail with a diagnostic, unless a requiredName has been specified for that dependency.
OPTIONALInjects one dependency into the dependent serviceInjects nullptr into the dependent service. td>Publication will fail with a diagnostic, unless a requiredName has been specified for that dependency.
NInjects all dependencies of the dependency-type that have been registered into the dependent service, using a QList Injects an empty QList into the dependent service. See 'Normal behaviour'
PRIVATE_COPYInjects a newly created instance of the dependency-type and sets its QObject::parent() to the dependent service. If the dependency-type has an accessible default-constructor, this will be used to create an instance of that type.
If no default-constructor exists, publication of the ApplicationContext will fail. 		Publication will fail with a diagnostic, unless a requiredName has been specified for that dependency.
Enumerator
MANDATORY 

This dependency must be present in the ApplicationContext.

OPTIONAL 

This dependency need not be present in the ApplicationContext. If not, nullptr will be provided.

N 

All Objects with the required service_type will be pushed into QList and provided to the constructor of the service that depends on them.

PRIVATE_COPY 

This dependency must be present in the ApplicationContext. A copy will be made and provided to the constructor of the service that depends on them. This copy will not be published in the ApplicationContext. After construction, the QObject::parent() of the dependency will be set to the service that owns it.

Specifies a constructor-argument that shall be resolved by the QApplicationContext. Use this function to supply resolvable arguments to the constructor of a Service. The result of resolving the placeholder must be a String that is convertible via QVariant::value<T>() to the desired type.

Example

Service<QIODevice,QFile> service{ resolve("${filename:readme.txt}") };
Parameters

Specifies a constructor-argument that shall be resolved by the QApplicationContext. The result of resolving the placeholder must be a String that is convertible via QVariant::value<T>() to the desired type.
Note: The expression is allowed to specify embedded default-values using the format ${identifier:defaultValue}. However, this does not make much sense, as it would render the parameter defaultValue useless, since the embedded default-value would always take precedence!

Example

Service<QIODevice,QFile> service{ resolve("${filename}", QString{"readme.txt"}) };
Parameters
expressionmay contain placeholders in the format ${identifier}.
defaultValuethe value to use if the placeholder cannot be resolved.
Returns
a Resolvable instance for the supplied type.
